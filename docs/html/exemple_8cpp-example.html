KinectInterface  v0.1
Créer une interface facilitant l'interprétation des données envoyées par la Kinect
exemple.cpp

Principe

Ce module est pensé comme une surcouche à la NUI et doit servir d'interface entre une application et le capteur Kinect. Le centre du module est la classe KinectInterface. Cette dernière doit encapsuler tout ce qui est nécessaire à l'utilisation de ce dernier par l'application de sorte à :

  1. ne pas encombrer le code gérant la logique de l'application
  2. offrir une assurance qu'aucune erreur n'a eu lieu lors de l'initialisation sans qu'il soit nécessaire de vérifier chaque appel dans l'application. Le but est donc de simplifier l'interaction avec la Kinect et de procurer une interface facile d'utilisation aux applications l'utilisant, diminuant ainsi le temps de développement desdites applications.

Le fonctionnement prévu est le suivant : la classe KinectInterface sera instanciée au début du programme et initialisera correctement la Kinect en fonction des arguments passés à son constructeur. Lorsque sa méthode start() sera appelée, elle commencera à recevoir les données envoyées par la Kinect et à les traiter, générant des évènements qui seront ensuite récupérés par l'application via la méthode pollEvent(). Ces évènements sont des instances de la classe KinectEvent, prévue pour rassembler tout les types de données de retour possibles dans une structure dont le traitement sera ensuite aisé.

Fonctionnalités

En construction (le but à court terme est la détection complète de squelettes (articulations et tout ce qui peut en être déduit). Voir la liste des choses à faire.

Exemple d'utilisation

Cet exemple montre comment il est possible d'utiliser le module KinectInterface pour détecter des évènements indiquant le déplacement du squelette et l'afficher dans une fenêtre SFML. On remarque que le traitement des informations est en effet quasiment invisible.

#include "SFML/Graphics.hpp"
#include "Kinect.hpp"
int main()
{
//initialisation de l'objet KinectInterface en lui donnant ce que l'on veut traquer
KtI::KinectInterface KInterface(KtI::INITIALIZE_SKELETON | KtI::SKELETON_NEAR_MODE);
sf::RenderWindow window(sf::VideoMode(640, 480), "KinectEvent test");
//un tableau de cercles représentant notre squelette
sf::CircleShape articulations[KtI::SKELETON_JOINTS_COUNT];
for (int i = 0; i < 20; i++)
{
articulations[i].setFillColor(sf::Color::Red);
articulations[i].setRadius(8);
}
//on lance l'acquisition des trames de données provenant de la Kinect
KInterface.start();
while (window.isOpen())
{
//déclaration d'un évènement KinectEvent
KtI::KinectEvent kEvent;
//boucle d'évènement de la Kinect
while (KInterface.pollEvent(kEvent))
{
//si on détecte un mouvement du squelette, on met à jour ses positions
if (kEvent.type == KtI::KinectEvent::SkeletonMoved)
{
for (int i = 0; i < KtI::SKELETON_JOINTS_COUNT; i++)
{
sf::Vector2f pos;
pos.x = kEvent.skeleton.positions[i].x;
pos.y = kEvent.skeleton.positions[i].y;
articulations[i].setPosition(pos);
}
}
}
sf::Event event;
while (window.pollEvent(event))
{
if (event.type == sf::Event::Closed)
window.close();
}
window.clear();
for (int i = 0; i < KtI::SKELETON_JOINTS_COUNT; i++)
{
window.draw(articulations[i]);
}
window.display();
}
return 0;
}
